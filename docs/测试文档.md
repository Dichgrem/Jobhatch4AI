# 测试文档

## 运行

```bash
nix develop            # 进入开发环境
just test              # 运行全部测试
```

或手动：

```bash
cd src/backend
python3 -m pytest tests/ -v               # 全部
python3 -m pytest tests/ -k "not Chat"    # 跳过需 API key 的测试
```

## 结构

```
tests/
├── conftest.py           # sys.path 设置
├── test_auth.py          # 用户认证（11 个）
├── test_preprocess.py    # 分词/词表/BOW/术语（15 个）
├── test_classifier.py    # MLP 分类器/训练（8 个）
├── test_recommender.py   # 技能推荐/余弦相似度（7 个）
├── test_rag.py           # RAG 提示构建/文档加载（5 个）
├── test_pipeline.py      # CSV 管道/统计（5 个）
└── test_api.py           # HTTP 端到端（14 个）
```

## 覆盖范围

| 模块 | 测试数 | 内容 |
|------|--------|------|
| Auth | 11 | 密码哈希、注册/登录/登出、token 管理、users.json 持久化 |
| Preprocess | 15 | jieba 分词（中文/空串/单字过滤/大小写）、词表构建（min_freq/max_size）、BOW 向量化、技术术语提取 |
| Classifier | 8 | MLP 初始化/前向传播/预测、训练收敛性、eval 模式确定性、技能分类统计 |
| Recommender | 7 | 技能推荐（基础/全覆盖/空输入）、余弦相似度（相同/正交/批量/范围） |
| RAG | 5 | 提示构建（单块/多块/空块）、文档目录加载 |
| Pipeline | 5 | CSV 解析（薪资/学历/经验统计）、状态存取、缺失列容错 |
| API | 11+3 | 健康检查、注册/登录/重复检测/错误密码、数据摘要/薪资/学历/分析/分类、聊天上下文 |

> Chat SSE 端点（3 个）标记为 skip，需要 `ARK_API_KEY` 环境变量。

## 测试模式

- **单元测试**：纯函数测试，不依赖外部服务（auth, preprocess, recommender, rag）
- **模型测试**：PyTorch MLP 结构验证 + 小样本训练（classifier）
- **集成测试**：FastAPI TestClient 模拟 HTTP 请求（api）
- **管道测试**：CSV → pipeline.run() → 输出结构验证（pipeline）
